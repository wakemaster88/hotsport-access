Neuen Raspberry Pi einrichten

Diese Anleitung führt dich in fünf Schritten durch die Inbetriebnahme eines neuen Drehkreuz-Pis. Geplante Dauer: ~10 Minuten pro Pi (zzgl. Zeit für das Image-Flashen).

Material & Voraussetzungen

  • Raspberry Pi (3B+, 4 oder Zero 2 W) mit microSD-Karte (min. 8 GB)
  • Netzteil (offizielles Pi-Netzteil empfohlen) und LAN- oder WLAN-Verbindung ins selbe Netz wie der Hub
  • Scanner-Hardware: USB-QR-/RFID-Wedge-Scanner, USB-Kamera oder MFRC522-RFID-Modul (siehe Schritt 5)
  • Relais-Modul (5 V, aktiv-high) am GPIO 24 und Buzzer am GPIO 23 (Standard, beides im Dashboard pro Pi änderbar)
  • Den Hub-Pi-Token – findest du oben in dieser Anleitung (Quelle: HOTSPORT_HUB_PI_TOKEN in /etc/hotsport-hub.env).
  • Internet auf dem Pi – nur beim ersten Bootstrap. Der Installer zieht ~30 MB Pakete (apt + pip + git clone von GitHub). Nach Schritt 2 kann der Pi dauerhaft offline bleiben; alle weiteren Updates kommen vom Hub.
1

Raspberry Pi OS Lite flashen

Lade den Raspberry Pi Imager und flashe Raspberry Pi OS Lite (64-bit) auf die SD-Karte. Im Imager unter „OS-Anpassungen" bitte folgendes setzen:

  • Hostname: hotsport-pi-NN (z.B. hotsport-pi-01)
  • SSH aktivieren – mit Passwort oder Public-Key
  • Benutzer: pi oder ein eigener Admin-User
  • WLAN/Ethernet-Konfig falls nicht per LAN angeschlossen
  • Locale & Tastatur: Deutsch

Tipp: Nimm für jeden Pi einen eindeutigen Hostnamen. Der Hub nutzt diesen als Default-pi_id, falls du keine eigene setzt.

2

Pi-App via GitHub installieren

SSH auf den Pi. Der Bootstrap erfolgt direkt aus dem öffentlichen GitHub-Repo.

Wichtig: Der Pi braucht für diesen einen Schritt Internet (apt + pip + git, ~30 MB). Danach kann er für immer offline bleiben – alle weiteren Updates kommen vom Hub im LAN.

2a) Repo klonen und Installer ausführen

sudo apt-get update
sudo apt-get install -y git
git clone https://github.com/wakemaster88/hotsport-access.git
cd hotsport-access/pi-app
sudo bash scripts/install.sh keyboard

Letztes Argument ist der Reader-Modus: keyboard (USB-Wedge, Standard), qr_camera oder rfid_mfrc522.

2b) Bootstrap-Konfiguration eintragen

Hub-URL und Token sind bereits eingesetzt. Nur pi_id, name und location auf den jeweiligen Pi anpassen. 

sudo nano /etc/hotsport-access/config.toml
pi_id     = "hotsport-pi-01"
name      = "Eingang Nord"
location  = "hotsport"

state_dir = "/var/lib/hotsport-access"
health_bind_host = "127.0.0.1"
health_bind_port = 8765

[hub]
base_url                      = "{{ hub_url }}"
pi_token                      = "{% if pi_token %}{{ pi_token }}{% else %}BITTE-EINTRAGEN{% endif %}"
heartbeat_interval_seconds    = 5.0
update_check_interval_seconds = 30.0
{% if not pi_token %}

Hinweis Hub läuft ohne HOTSPORT_HUB_PI_TOKEN. Token in /etc/hotsport-hub.env setzen, Hub neu starten, danach hier neu laden.

{% endif %}

2c) Service starten

sudo systemctl restart hotsport-access hotsport-updater
sudo systemctl status hotsport-access --no-pager -l | head -20
journalctl -u hotsport-access -n 30 --no-pager

Innerhalb von ~5 Sekunden taucht der Pi unten im Dashboard auf. Ab jetzt läuft alles über den Hub – Pi kann vom Internet getrennt werden.

3

Pi im Dashboard konfigurieren

Im Dashboard beim neuen Pi auf das ▾ klicken und in der rechten Spalte folgendes setzen:

Interface-ID
Die resourceId aus der Binarytec-API (z.B. 101).
Richtung
in (Eingang) oder out (Ausgang).
Reader-Modus
keyboard – USB-Wedge-Scanner (Standard, einfachste Variante)
qr_camera – USB-Kamera mit OpenCV
rfid_mfrc522 – SPI-RFID-Modul
Reader-Devicepfad / Kamera-Index
Bei Wedge-Scannern /dev/input/event0 (mit ls /dev/input/by-id/ auf dem Pi prüfen). Bei Kameras meist 0.
GPIO-Pins
Standard: Relais GPIO 24, Buzzer GPIO 23, Pulsdauer 1.0 s. Nur ändern, wenn die Verkabelung anders ist.

Nach Speichern ändert sich der Config-Fingerprint. Der Pi erkennt das beim nächsten Heartbeat (≤ 5 s) und startet sich automatisch mit der neuen Konfig neu – kein SSH nötig.

4

Reader-spezifische Hinweise

--mode beim Bootstrap entscheidet, welche Pakete der Installer mit-installiert. Wenn du den Modus später wechseln willst, reicht meistens das Setzen im Dashboard plus das Nachinstallieren der Reader-Pakete:

USB-Kamera (qr_camera): 

sudo apt-get install -y python3-opencv
sudo /opt/hotsport-access/venv/bin/pip install \
    -r /opt/hotsport-access/current/requirements-camera.txt
sudo systemctl restart hotsport-access

MFRC522-RFID (SPI): Einmalig sudo raspi-config → Interface Options → SPI aktivieren, dann: 

sudo apt-get install -y python3-spidev
sudo /opt/hotsport-access/venv/bin/pip install \
    -r /opt/hotsport-access/current/requirements-rfid.txt
sudo systemctl restart hotsport-access

Troubleshooting

Pi taucht nicht im Dashboard auf
  • Auf dem Pi: sudo systemctl status hotsport-access – Dienst aktiv?
  • Logs: journalctl -u hotsport-access -n 50 --no-pager
  • Vom Pi aus den Hub testen: curl -i {{ hub_url }}/health
  • Token korrekt? In /etc/hotsport-access/config.toml muss pi_token exakt dem HOTSPORT_HUB_PI_TOKEN auf dem Hub entsprechen.
Bootstrap bricht ab beim apt-get install

Der Pi braucht für Schritt 2 Internet. Test: ping -c 2 8.8.8.8 und curl -I https://github.com. Falls beides fehlschlägt: Pi temporär ans normale WLAN hängen (sudo raspi-config → System Options → Wireless LAN), Bootstrap durchlaufen lassen, danach zurück ins Hub-Netz.

Code-Update auf einem bereits installierten Pi

Im Normalfall kommen Updates vom Hub. Falls du den Pi-App-Code manuell aus GitHub aktualisieren willst (z.B. nach main-Branch-Änderungen): 

cd ~/hotsport-access && git pull
cd pi-app && sudo bash scripts/install.sh keyboard
sudo systemctl restart hotsport-access

config.toml, venv und Daten bleiben erhalten – der Installer ist idempotent.

Status bleibt „unkonfiguriert"

Im Pi-Detailpanel (▾) muss eine interface_id gesetzt und gespeichert werden. Erst dann ist die Live-Config vollständig und der Pi geht in den Bereit-Status.

Reader liest nichts ein
  • Wedge-Scanner: ls /dev/input/by-id/ – den passenden Pfad als Reader-Devicepfad ins Dashboard eintragen.
  • Kamera: v4l2-ctl --list-devices auf dem Pi installieren und richtigen Index setzen.
  • RFID: SPI in raspi-config aktiv? User hotsport ist Mitglied der Gruppe spi?
Drehkreuz öffnet nicht

Im Dashboard prüfen: ist der Pi aktiv? Stimmt der Relais-GPIO? Test direkt auf dem Pi: 

sudo /opt/hotsport-access/venv/bin/python -c \
  "from gpiozero import OutputDevice; import time; \
   r = OutputDevice(24); r.on(); time.sleep(1); r.off()"
